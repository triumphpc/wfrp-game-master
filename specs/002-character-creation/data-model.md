# Data Model: Улучшение создания персонажей WFRP

## Entity: Character

Расширение существующей модели персонажа для поддержки русских характеристик.

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| ID | string | Да | Уникальный идентификатор |
| Name | string | Да | Имя персонажа |
| RussianStats | map[string]int | Да | Характеристики на русском: ББ, ДБ, СС, И, Л, О, СТ, К |
| OriginalStats | map[string]int | Нет | Оригинальные WFRP характеристики |
| Career | string | Да | Профессия |
| Skills | []string | Нет | Навыки |
| Status | string | Нет | Статус |
| Sheet | string | Да | Полная карточка в Markdown |

## Russian Stats Mapping

| Русское | Английское | Описание |
|---------|------------|----------|
| ББ | WS | Боевая Пригодность |
| ДБ | BS | Дистанция Боя |
| СС | S | Сила |
| И | I | Инициатива |
| Л | Ag | Ловкость |
| О | WP | Общение |
| СТ | Fel | Стойкость |
| К | T | Классовая (Выносливость) |

## Entity: CharacterCreationState (расширение)

Добавить флаг для поддержки вопросов к LLM:

| Поле | Тип | Описание |
|------|-----|----------|
| State | CharacterCreationState | Текущий этап |
| IsAskingLLM | bool | Флаг - пользователь задаёт вопрос |
| LLMQuestion | string | Вопрос к LLM |

## API Contracts

### Telegram Commands

```
/character [имя]  - Начать создание персонажа (или показать справку)
/characters        - Показать список всех персонажей
сгенери имя       - Сгенерировать имя через LLM
сгенери сам       - Синоним "сгенери имя"
```

### Character Creation Flow

```
Начало (/character Имя)
    ↓
Этап расы → ? → Запрос к LLM
    ↓
Этап карьеры → ? → Запрос к LLM
    ↓
Этап характеристик → ? → Запрос к LLM
    ↓
... (существующие этапы)
    ↓
Карточка персонажа (полная)
```

## Validation Rules

- Имя персонажа: 2-50 символов
- Характеристики: 0-100 (без учёта advances)
- Уникальность имени в рамках кампании
